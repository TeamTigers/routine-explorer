Technology Used in the Application


  • JSP
  • Servlet
  • Server

JSP stands for 'Java Server Page'. JSP pages use several delimiters for scripting functions. The most basic is "< % ... %>", which encloses a JSP scriptlet. A scriptlet is a fragment of Java code that is run when the user requests the page. It can compile scripts like JS but outside of html tag. It can also connect with database like SQL-Server, SQLite etc. Any script should be written outside of the html tag.

A Java servlet is a Java program that extends the capabilities of a server. A Java servlet processes or stores a Java class in Java EE that conforms to the Java Servlet API. It can integrate with web toolkit but it needs the support of PrintWriter class built in Java EE to display any information. It has the support of connecting database also. It is mandatory to use web.xml to pass request from jsp to servlet.

Apache Tomcat, often referred to as Tomcat Server, is an open-source Java Servlet Container developed by the Apache Software Foundation (ASF). Tomcat implements several Java EE specifications including Java Servlet, JavaServer Pages (JSP), Java EL, and WebSocket, and provides a "pure Java" HTTP web server environment in which Java code can run. It is better to use as a built in plugin provided by IDE.

  • JavaScript
  • Framework
  • Database

JavaScript enables interactive web pages and thus is an essential part of web applications. The vast majority of websites use it, and all major web browsers have a dedicated JavaScript engine to execute it. We have used jQuery and AJAX using the JavaScript concept to for initiating dynamic fields and User comfort. All these scripts are integrated with both JSP and Servlet classes.

In this Web-Application we have used 'Material-Kit CSS' that has been developed by a famous web-design organization known as 'Creative Tim'. We have also used Materialize CSS owned by the 'Tech-Giant GOOGLE', which was developed by some Japanese web-developers. We haves used this for a responsive UI and better user experience from this web-application.

Previously we have used Microsoft SQLServer 2017, the latest version of SQLServer. But for better user experience and to make our project lite-weight we have used SQLite database. For SQLite support in Java we have used SQLite JDBC (Java Database Connection). For browsing database we have used Mozilla FireFox Extension (version V.52) known as 'SQLite Manager'.

How the app work

This app is basically used for getting the routine from the excel file provided from the department of Software Engineering at the beginning of every semester. Besides getting routine this app also helps to find about ten types of bugs from the routine for teachers and students integreting routine and course offer excel file. First of all admin needs to upload both routine file and course offer file in the system. After uploading files, admin will publish the routine version. Teachers and students then get their routine from the system of the published routine version. Students can get their routine by entering their seemster name and section. Teachers can get their routine by entering their Initial. Teachers and students can get their routine as pdf from the system.

What problem it solves

This app helps to find about ten types of bugs from the routine for teachers and students integreting routine and course offer excel file. It generates two report of bugs for teachers and students. For teacher's report it generates bugs like teachers having continuous three or more classes, double class in a single slot, classes in both main campus and permaent campus, class starts at 8:30 am and 4:00 pm etc. For students it generates bug like students having more than three classes without any break, classes from 8:30 am to 4:00 pm, single class in day, class break more than 1:30 hours etc.




Institutions demand

This project is basically designed for the department of Software Engineering of Daffodil International University. But other educational instutions can also use this system if they maintain the same process to publish routine versions. This project is well documented and every function is easily understandable for the coder. So if any institution wants to use this system for themselves they can also edit the code as their need.
